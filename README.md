# SponsorCheck

SponsorCheck is a UK Skilled Worker SOC code and salary route checker designed for professional use.

A user pastes or uploads a job description. The app suggests likely Standard Occupational Classification (SOC) codes from local GOV.UK datasets, shows why the match was chosen, checks standard salary and possible lower salary routes, asks for missing facts, and generates a downloadable PDF report.

## ⚠️ Disclaimer

SponsorCheck gives a general indication of possible SOC codes and salary routes based on the information provided. It does not make immigration decisions, confirm visa eligibility, or replace GOV.UK guidance. Always check the latest GOV.UK Skilled Worker pages before relying on the result.

**SponsorCheck is not legal advice, immigration advice, sponsor compliance advice, or a replacement for GOV.UK.**

## Core Principles

- **Local Data Only:** The app uses local JSON files during normal runtime. It does not scrape GOV.UK dynamically.
- **Controlled AI Classification:** The LLM only classifies job descriptions into SOC codes from a validated local shortlist generated by TF-IDF retrieval. It does not invent SOC codes.
- **Deterministic Salary Engine:** The LLM does not calculate salary or decide visa eligibility. All salary routes and checks are processed via deterministic Python code based on the local rules datasets.

## Recent Updates & Features

- **Dynamic Fact Injection:** The AI prompt dynamically reads minimum salary thresholds (e.g., £41,700 for Standard, £33,400 for discounts) from the dataset, strictly instructing the LLM to avoid hallucinating outdated figures and preventing salary numbers from biasing SOC code decisions.
- **Robust Error Surfacing:** Automatically captures LLM/Ollama API connection errors or hallucinated JSON payloads and displays the exact reason for the failure in the UI alongside a fallback to the rule-based lexical matching engine.
- **Hot-Reloading Docker:** Both the FastAPI backend and Next.js frontend are configured to support live hot-reloading inside Docker Compose via volume mounts.
- **PDF Encoding Fixes:** Fixed `wkhtmltopdf` UTF-8 encoding issues to correctly render the £ symbol on generated PDF reports.
- **UI Enhancements:** Resolved CSS specificity conflicts between Tailwind utility classes and global styles for better visual hierarchy and readability on the SOC decision cards.

## Project Structure

- `frontend/`: Next.js, Tailwind CSS, TypeScript frontend with component-based architecture.
- `backend/`: FastAPI, Pydantic v2, Python backend for retrieval, LLM interaction, and salary deterministic logic.
- `data/`: Cleaned drop-in JSON files representing the GOV.UK SOC data, special datasets, and rule schemas.

## Startup Guide

### Option A: Docker (Recommended)

You can run both the backend and frontend using Docker Compose. Hot-reloading is fully enabled for both environments.

1. Ensure Docker Desktop is running.
2. Open your terminal in the root project directory.
3. Run the following command:

```bash
docker compose up --build
```

- The frontend will be available at **http://localhost:3000**.
- The backend API will be available at **http://localhost:8000**.

*Note: The LLM provider (e.g., Ollama or LM Studio) should be run separately on your host machine. Docker will connect to it via your network or `host.docker.internal`.*

### Option B: Local Development

If you prefer to run the applications natively without Docker:

**Requirements:** Node.js 18+ (frontend) and Python 3.9+ (backend)

#### 1. Backend

Navigate to the `backend` directory, create a virtual environment, and start the FastAPI server:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Or .venv\Scripts\activate on Windows
pip install -e .[dev]
uvicorn sponsorcheck.api.main:app --reload
```

#### 2. Frontend

Open a new terminal, navigate to the `frontend` directory, install dependencies, and start the Next.js development server:

```bash
cd frontend
npm install
npm run dev
```

The frontend will be available at **http://localhost:3000**.

## Provider Configuration

SponsorCheck supports multiple classification backends. You can configure these in the UI's settings panel or via environment variables in `.env` (see `.env.example`).

### Rule-Based Mode (Default / No Model)
```env
SPONSORCHECK_PROVIDER=rule_based
```

### Ollama Mode
```env
SPONSORCHECK_PROVIDER=openai_compatible
LLM_BASE_URL=http://localhost:11434/v1
LLM_API_KEY=ollama
LLM_MODEL=gemma4:e2b
```

### LM Studio Mode
```env
SPONSORCHECK_PROVIDER=openai_compatible
LLM_BASE_URL=http://localhost:1234/v1
LLM_API_KEY=lm-studio
LLM_MODEL=local-model
```

### OpenAI Compatible Endpoint
You can use any OpenAI-compatible API endpoint by providing the appropriate base URL and API key.

## Reports

SponsorCheck generates HTML and downloadable PDF reports containing the full evaluation. These are built directly on the backend using Jinja2 and `pdfkit`.

## Tests

To run the backend test suite, navigate to the `backend` directory and run:

```bash
pytest tests/
```
